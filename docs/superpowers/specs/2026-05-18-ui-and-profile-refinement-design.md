# 디자인 명세서: UI 비율 최적화 및 프로필 기능 강화

## 1. 개요
탐색(Discover) 페이지의 유저 리스트 아이템 비율을 최적화하여 가독성과 밀도를 높이고, 프로필 페이지에 '책으로 보기' 기능을 추가하여 자서전 감상 경험을 홈 화면과 통일한다. 또한, 타인 프로필 방문 시 개인정보 보호를 위해 팔로워 명단 노출을 제한한다.

## 2. 변경 사항

### 2.1 탐색(Discover) 페이지 리스트 아이템 최적화
- **목적**: 유저 검색 결과 리스트의 과도한 세로 점유율을 줄이고 시각적 버그 수정.
- **수정 사항**:
    - 리스트 아이템(`div`) 패딩: `p-8` -> `p-5` (세로폭 축소)
    - 아바타 컨테이너 높이: `h-124` -> `h-16` (오타 수정 및 비율 정상화)
    - 곡률(Border Radius): `rounded-[40px]` -> `rounded-3xl`
    - 레이아웃: 좌측(아바타, 유저네임), 우측(프로필 보기 아이콘 또는 버튼) 형태의 균형 잡힌 배치.

### 2.2 프로필 페이지 자서전 감상 기능 (공통)
- **목적**: 홈 화면의 '책/목록' 뷰 경험을 프로필 페이지로 확장.
- **수정 사항**:
    - `viewMode` 상태(`'book' | 'list'`) 추가 (기본값: `'book'`).
    - 홈 화면과 동일한 스타일의 '뷰 전환 토글' UI를 프로필 헤더 하단에 배치.
    - `records` 데이터 렌더링 시 선택된 `viewMode`에 따라 `BookViewer` 또는 `ListViewer` 호출.

### 2.3 프로필 접근 권한 및 개인정보 보호
- **본인 프로필 (`isOwnProfile === true`)**:
    - 팔로워/팔로잉 숫자 표시.
    - 기존처럼 하단에 팔로워 명단(`RelationshipTabs`)을 노출하여 관리 가능.
- **타인 프로필 (`isOwnProfile === false`)**:
    - 팔로워/팔로잉 **숫자만** 표시.
    - 팔로워 명단(`RelationshipTabs`)을 숨김 처리하여 다른 유저의 인맥 정보를 보호.
    - 오직 해당 유저가 작성한 자서전 기록(공개 범위 내)만 감상 가능.

## 3. 기술적 고려 사항
- **컴포넌트 재사용**: `src/components/records/BookViewer.tsx`와 `ListViewer.tsx`를 프로필 페이지에서도 그대로 사용.
- **상태 관리**: `viewMode`는 로컬 state로 관리하며, 페이지 진입 시 해당 유저의 기록을 먼저 fetch한 후 뷰어에 전달.
- **UI 일관성**: 토글 버튼의 스타일(색상, 곡률, 그림자)을 홈 화면(`src/app/page.tsx`)과 100% 일관되게 유지.

## 4. 성공 기준
- 탐색 페이지에서 검색 결과가 한 화면에 더 많이 노출됨 (최소 3~4명 이상).
- 다른 유저 프로필에서 '책으로 보기'를 통해 실제 책을 넘기는 듯한 경험이 정상 작동함.
- 타인 프로필 하단에 더 이상 팔로워 명단 리스트가 보이지 않음.
